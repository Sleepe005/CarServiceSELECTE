# CarServiceSELECTE

## Подключение к серверу
### 1. Создание SSH ключа (инструкция для Linux/Mac, Bash в Windows)
`ssh-keygen -t ed25519`

На экране появится сообщение с предложением выбрать путь для сохранения файлов:

`Enter file in which to save the key (/home/user/.ssh/id_rsa):`

Здесь достаточно нажать клавишу Enter, чтобы оставить путь по умолчанию — скрытый каталог .ssh в домашней директории пользователя.

Далее система предложит ввести кодовую фразу для дополнительной защиты самого файла ключа — на случай, если к локальному компьютеру кто‑то посторонний все же получит физический доступ (не обязательно)

После завершения процесса терминал выведет сообщение об успешном сохранении, а также цифровую подпись (fingerprint) и ее графическое представление.

После успешной генерации SSH-пары на компьютере появляются два математически связанных файла — закрытый приватный и открытый публичный ключи — в том каталоге, который мы указали, то есть ~/.ssh/ по умолчанию.

Чтобы вывести содержимое открытого ключа на экран, используется команда: `cat ~/.ssh/id_rsa.pub`

### 2. Подключение к серверу по SSH
`ssh <user>@93.92.206.106 -p <port>` 

После подключения выполнить `apk add docker-cli` для установки Docker CLI в случае перезагрузки сервера

Для подключения к базе данных PostgreSQL `docker exec -it project-postgres psql -U <bdName> -d <bdPassword>`

### 3. Подключение к БД по SSH
Подключение происходит при помощи создания SSH тунеля с базой данных на сервере
`ssh -L 5433:postgres:5432 <user>@93.92.206.106 -p <port>`

Тунель активируется, треминал должен зависнуть. Тунель работает только если открыт терминал

Для подключения к бд на сервере через psql или ide можно использовать localhost:5433
